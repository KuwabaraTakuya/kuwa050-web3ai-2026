# VPC v4 - Kuwa050

## V4 → V5 で変わった部分

- **目的選択ボトムシートを追加**：▶ 開始ボタンを押したとき、SNSを開く理由（暇つぶし／情報収集／友達チェック／なんとなく）を選ぶシートがスライドアップ表示されるようにした。V4は開始と同時にタイマーが始まるだけだった
- **スキップ選択肢を設けた**：目的を選びたくないときは「スキップ（記録しない）」でそのまま開始できるようにした。強制入力にするとかえって使われなくなるため
- **セッションログに目的バッジを追加**：ログ一覧の各エントリに、選んだ目的を色付きタグで表示するようにした。「いつ・何分」に加えて「なぜ」が一目でわかる
- **目的別ブレークダウンカードをログタブに追加**：今日の利用を目的別に集計し、件数・分数のバー表示と割合グラフをログタブ上部に表示するようにした
- **インサイトメッセージを自動生成**：最も多い目的に応じたメッセージ（「なんとなくが○%を占めています」等）を表示し、翌日の行動変容を促すようにした
- **localStorageキーを `sns-v5-{id}` に変更**し、V4データとの競合をなくした

## V4を見直してみてのフィードバック

- 「記録はできているが、なぜ使い過ぎたかが見えないので次に活かせない」→ 時間と回数は可視化できていたが、利用動機が記録されていないため振り返りが「また使いすぎた」で終わっていた
- 「開始ボタンを押す前に一度立ち止まる仕組みがない」→ タイマーが即座に始まるため、無意識にSNSを開く習慣を断ち切るトリガーがなかった
- 「ログを見ても何が原因かわからないので改善しようがない」→ 使用時間のデータだけでは「どうすればよかったか」に結びつかない
- 「暇つぶしでXを開く自分に気づいていなかった」→ 動機を選ぶことで初めて自分の行動パターンを自覚できることがわかった
- 「インサイトが文字だけでは読まない」→ 目的別バーと組み合わせることで視覚的に自分ごと化しやすくなる

## フィードバックからの気づき

- **「記録する」と「理解する」は別の行為**：時間を記録しても動機が抜けていると、数字の羅列にしかならない。「なぜ開いたか」という1ステップを加えることで、データが行動変容につながる情報になる
- **摩擦は敵ではなく道具になる**：開始直前に目的を選ばせる「小さな摩擦」は、無意識の利用を意識的な選択に変える介入点として機能する。摩擦の有無ではなく、摩擦を「どこに置くか」が設計の核心だった
- **振り返りは感情ではなくデータで促す**：「使いすぎた」という後悔だけでは行動は変わらない。「なんとなく開いた割合が70%」という具体的な数字が、翌日の意図的な行動変容のきっかけになる
